How Chunking is Done

First of all, we created a model (sometimes called database schema) that accept the user’s information for signup and also the model for receiving the file which needs to chunked. For each file chunked it is referenced to the user itself to prevent other users from seeing it. Files not saved will be deleted after the speculated time given. The process of chunking a file goes like this.

  • The User import the file into the dropbox, with the maximum row number the user want in a file.
  • The store file function reads through the imported file, split it by the maximum row number into each new file created then saves all the chunked files in a zip folder.
  • If the user decides to save the file, it goes into the media folder.
  • If not saved, it goes into the temporary folder and get deleted after 15 mins of creation.
  • For the user to review just one file, a standalone file will be created to see how it’s being chunked

View how the chunking works on our Github readme
